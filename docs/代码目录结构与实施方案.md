## 2. 各模块职责

- `scripts/prepare_dataset.py`：把区域标注转换为 `interior_mask` 和 `seed_heatmap`（距离变换归一化）。
- `src/models/unet_dual_head.py`：实现双头 U-Net，输出 `interior_pred` 与 `seed_pred`。
- `src/losses/dice_bce.py`：`Interior Loss = BCE + Dice`。
- `src/losses/weighted_mse.py`：`Seed Loss = Weighted MSE`，权重 `w=1+αH`。
- `src/postprocess/*`：实现平滑、局部峰值提取、watershed、连通域修正、小区域过滤。
- `src/metrics/*`：实现 IoU、Dice、Heatmap MSE、Region-level mIoU、欠/过分割率、Leakage Rate。

## 3. 实施方案（分阶段）

### 阶段 A：数据与标签（第 1 周）

1. 明确标注格式（区域 ID 图或多 mask）。
2. 完成 `prepare_dataset.py`：
   - 生成 `interior_mask.png`；
   - 逐区域 distance transform；
   - 合并得到 `seed_heatmap.png`。
3. 产出 train/val/test 划分清单。

**交付物**：`data/processed/*` 可直接训练的数据。

### 阶段 B：模型与损失（第 2 周）

1. 完成 `unet_dual_head.py`。
2. 完成 `dice_bce.py` 与 `weighted_mse.py`。
3. 在 `trainer.py` 打通 forward/loss/backward/ckpt。

**建议初始超参**：`lr=1e-3, batch=8, epoch=30, input=256, λ1=1.0, λ2=1.0, α=3.0`。

**交付物**：可稳定训练并输出 loss 曲线。

### 阶段 C：后处理与可视化（第 3 周）

1. `peak_extract.py`：heatmap blur + local maxima。
2. `watershed.py`：基于 `interior_bin` 与 markers 进行分割。
3. `refine.py`：连通域修正 + 小区域过滤。
4. `visualize.py`：输出 `region_label_map`、单区域 mask、随机平涂图。

**交付物**：单张图可得到可填色区域分割结果。

### 阶段 D：评测与对比实验（第 4 周）

1. 实现 baseline1（传统形态学）与 baseline2（仅 interior）。
2. 实现完整指标评估。
3. 做两个消融：
   - Binary Seed vs Seed Heatmap；
   - MSE vs Weighted MSE。

**交付物**：表格 + 可视化对比图。

## 4. 训练与推理命令约定

```bash
# 1) 数据准备
python scripts/prepare_dataset.py --config configs/default.yaml

# 2) 训练
python scripts/train.py --config configs/train_256.yaml

# 3) 验证/评估
python scripts/evaluate.py --config configs/train_256.yaml --ckpt outputs/checkpoints/best.pth

# 4) 推理
python scripts/infer.py --config configs/infer.yaml --input demo/lineart.png --ckpt outputs/checkpoints/best.pth
```

## 5. 关键风险与规避

1. **小区域丢失**：提高 `λ2` 和 `α`，并降低 `peak_threshold`。
2. **区域粘连**：调大 `min_distance`，并增加 watershed 前平滑。
3. **噪声 marker 过多**：提升 `peak_threshold` 或增加面积过滤。
4. **数据量偏小**：先 256 分辨率跑通，再逐步提升到 512。

## 6. 第一版里程碑（MVP）

- M1：数据生成脚本可用；
- M2：双头 U-Net 能训练并产出两张预测图；
- M3：后处理可生成 `region_label_map`；
- M4：Leakage Rate 相对 baseline2 明显下降。

> 建议先以“可跑通 + 可视化正确”为第一目标，再做指标最优化。
