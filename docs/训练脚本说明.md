# 训练脚本说明（`scripts/train.py`）

本文档介绍当前仓库训练脚本的用途、执行方式、配置项，以及模型结构（双头 U-Net）与训练流程。

## 1. 脚本定位

当前训练入口为：`scripts/train.py`。

其主要目标是：
- 快速验证阶段 B（双头分割 + seed 预测）训练循环是否能正常跑通；
- 串联模型、损失函数、优化器与日志打印；
- 为后续接入真实数据集（阶段 A）预留结构。

> 注意：当前版本默认使用 `DummyLineartDataset`（随机张量）作为占位数据，不代表真实训练效果。

---

## 2. 快速开始

### 2.1 依赖

脚本依赖以下核心库：
- `torch`
- `pyyaml`

### 2.2 运行命令

```bash
python scripts/train.py --config configs/train_256.yaml --device cpu
```

如果有可用 GPU，可替换为：

```bash
python scripts/train.py --config configs/train_256.yaml --device cuda
```

### 2.3 运行输出

训练过程中每隔 5 个 step 输出一次日志，例如：

```text
epoch=0 step=0 total=1.2345 interior=0.5678 seed=0.6667
```

字段含义：
- `total`: 总损失（加权和）
- `interior`: interior 分支损失
- `seed`: seed 分支损失

---

## 3. 配置文件说明（`configs/train_256.yaml`）

训练脚本通过 YAML 读取配置，核心区块如下：

- `model`
  - `in_channels`: 输入通道数（线稿通常为 `1`）
- `loss`
  - `lambda_interior`: interior 损失权重
  - `lambda_seed`: seed 损失权重
  - `alpha`: weighted MSE 的前景加权系数
- `optim`
  - `lr`: Adam 学习率
- `data`
  - `size`: 占位数据集样本数
  - `image_size`: 输入高宽（正方形）
  - `batch_size`: batch 大小
- `train`
  - `epochs`: 训练轮数

---

## 4. 模型结构介绍（`UNetDualHead`）

模型定义位于：`src/models/unet_dual_head.py`。

该模型是一个**共享编码器/解码器 + 双输出头**的 U-Net 变体，用于同时预测：
- `interior`：区域内部概率图（1 通道）
- `seed`：实例 seed 热力图（1 通道）

### 4.1 主干结构

- 输入：`[B, 1, H, W]`
- 编码器通道：`32 -> 64 -> 128 -> 256 -> 512`
- 解码器通道：`512 -> 256 -> 128 -> 64 -> 32`
- 跳连：标准 U-Net skip connection（编码器同尺度特征与解码器融合）

### 4.2 双头输出

在共享解码特征上，使用两个独立 `1x1 Conv + Sigmoid` 预测头：

1. `interior_head`
   - 输出 `interior_pred`，范围 `[0,1]`
2. `seed_head`
   - 输出 `seed_pred`，范围 `[0,1]`

这样设计的优势是：
- 主干特征共享，参数利用率高；
- 两个任务（区域与种子）可以协同学习；
- 推理阶段可直接为后处理（如 watershed）提供多路信息。

## 5. 训练流程说明

训练核心逻辑在 `src/engine/trainer.py`：

1. 从 batch 读取 `image / interior / seed` 并搬运到目标设备。
2. 前向得到 `interior_pred, seed_pred`。
3. 计算分项损失：
   - `loss_interior = interior_loss(interior_pred, interior_gt)`
   - `loss_seed = weighted_mse_loss(seed_pred, seed_gt, alpha)`
4. 总损失：

   \[
   L = \lambda_{interior} \cdot L_{interior} + \lambda_{seed} \cdot L_{seed}
   \]

5. 反向传播与参数更新（Adam）。
6. 返回并打印 `loss_total / loss_interior / loss_seed`。

## 6. 当前实现的边界与后续建议

### 6.1 当前边界

- 仅包含训练最小闭环与占位数据；
- 暂无 checkpoint 保存、验证集评估、学习率调度、混合精度等训练工程化能力；
- 日志输出为控制台打印，尚未接入 TensorBoard/W&B。

### 6.2 建议优先补充

1. 用真实 `LineartDataset` 替换 `DummyLineartDataset`；
2. 增加模型保存与断点续训；
3. 新增验证循环（val loss / 可视化指标）；
4. 视显存情况加入 AMP（`torch.cuda.amp`）；
5. 将训练参数统一收敛到配置文件并补充分层注释。
