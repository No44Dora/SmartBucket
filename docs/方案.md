下面是**改进后的完整方案**，把原来的二值 Seed Map 改成 **Seed Heatmap 回归**。整体思路变为：

> 输入非封闭黑白线稿 → U-Net 预测可填色区域内部图与区域中心热力图 → 从热力图提取 marker → watershed 分割 → 输出可快速填色的封闭区域。

------

# 1. 改进后的总体方案

## 模型

使用 **双头 U-Net**。

### 输入

```text
非封闭黑白线稿图
尺寸：256×256 或 512×512
通道：1 channel
```

### 输出

```text
Head 1：Region Interior Map
Head 2：Seed Heatmap
```

其中：

- **Region Interior Map**：预测哪些像素属于可填色区域内部；
- **Seed Heatmap**：预测每个区域的中心响应，区域中心最亮，靠近边界逐渐变暗。

------

# 2. 数据标签设计

## 2.1 Interior Map 真值

Interior Map 仍然是一个二值 mask：

```text
1：区域内部
0：线条 / 背景 / 不可填区域
```

例如脸部、头发块、衣服块、眼睛内部等区域都属于 interior。

------

## 2.2 Seed Heatmap 真值

不要再使用单像素 seed 点，而是为每个区域生成一个连续热力图。

推荐使用 **Distance Transform Heatmap**。

对于每个区域 mask：

```text
1. 取出该区域的二值 mask
2. 对该区域做 distance transform
3. 将距离图归一化到 [0, 1]
4. 合并所有区域的距离热力图
```

得到的 Seed Heatmap 具有以下特点：

```text
区域中心：接近 1
区域边缘：接近 0
区域外部：0
```

公式可以写成：

[
H_i(x,y)=\frac{D_i(x,y)}{\max(D_i)}
]

其中：

- (H_i(x,y))：第 (i) 个区域的 heatmap；
- (D_i(x,y))：像素 ((x,y)) 到该区域边界的距离；
- (\max(D_i))：该区域内最大距离。

最终 Seed Heatmap：

[
H(x,y)=\max_i H_i(x,y)
]

# 3. 改进后的网络结构

## 3.1 U-Net 主干

使用标准 U-Net：

```text
Encoder:
Conv Block → Downsample
Conv Block → Downsample
Conv Block → Downsample
Conv Block → Downsample

Decoder:
Upsample → Skip Connection → Conv Block
Upsample → Skip Connection → Conv Block
Upsample → Skip Connection → Conv Block
Upsample → Skip Connection → Conv Block
```

------

## 3.2 双输出头

最后分成两个 head：

```text
Interior Head:
1×1 Conv → Sigmoid → Interior Probability Map

Seed Heatmap Head:
1×1 Conv → Sigmoid → Seed Heatmap
```

两个输出都是：

```text
1 × H × W
```

# 4. 损失函数设计

## 4.1 Interior Loss

Interior 是二分类分割任务，使用：

[
L_{interior}=L_{BCE}+L_{Dice}
]

其中：

- BCE 负责像素级分类；
- Dice 负责区域重叠。

------

## 4.2 Seed Heatmap Loss

Seed Heatmap 是回归任务，推荐使用 **Weighted MSE Loss**。

普通 MSE：

[
L_{seed}=MSE(\hat{H},H)
]

但普通 MSE 容易让模型忽略中心高响应区域，所以建议使用加权 MSE：

[
L_{seed}=\frac{1}{N}\sum_{x,y}w(x,y)(\hat{H}(x,y)-H(x,y))^2
]

权重设计为：

[
w(x,y)=1+\alpha H(x,y)
]

推荐：

```text
α = 2 ~ 5
```

这样区域中心附近的误差会被更重视。

------

## 4.3 总损失

[
L=\lambda_1L_{interior}+\lambda_2L_{seed}
]

推荐初始设置：

```text
λ1 = 1.0
λ2 = 1.0
α = 3.0
```

如果 Seed Heatmap 学得偏弱，可以调整为：

```text
λ1 = 1.0
λ2 = 2.0
α = 5.0
```

------

# 5. 训练流程

## 5.1 输入数据

每条训练样本包括：

```text
input_lineart.png
interior_mask.png
seed_heatmap.png
```

------

## 5.2 训练步骤

```text
1. 读取非封闭线稿图
2. 读取 Interior Mask
3. 读取 Seed Heatmap
4. 输入 U-Net
5. 得到 Interior Prediction 和 Seed Heatmap Prediction
6. 计算 Interior Loss
7. 计算 Seed Heatmap Loss
8. 合并总损失
9. 反向传播
10. 更新模型参数
```

------

## 5.3 推荐训练参数

```text
Optimizer：Adam
Learning Rate：1e-3
Batch Size：8 或 16
Epoch：50 ~ 100
Input Size：256×256 起步
Scheduler：ReduceLROnPlateau / Cosine Annealing
```

如果你的数据较少，可以先用：

```text
Epoch：30
Input Size：256×256
Batch Size：8
```

先验证流程能跑通。

# 6. 推理与后处理流程

推理时，模型输出：

```text
Interior Probability Map
Seed Heatmap
```

然后进入后处理。

------

## 6.1 Interior Threshold

对 Interior Map 二值化：

```text
interior_bin = interior_pred > T_interior
```

推荐：

```text
T_interior = 0.5
```

如果区域断裂较多，可以降低到：

```text
T_interior = 0.4
```

------

## 6.2 Seed Heatmap 平滑

对预测 heatmap 做轻微 Gaussian blur：

```text
seed_smooth = gaussian_blur(seed_pred)
```

推荐：

```text
sigma = 1 ~ 2
```

这样可以减少噪声峰值。

------

## 6.3 Local Maxima 提取

从 Seed Heatmap 中提取局部峰值作为 watershed markers。

推荐参数：

```text
peak_threshold = 0.3 ~ 0.5
min_distance = 8 ~ 20
```

如果图像大小为 256×256：

```text
peak_threshold = 0.4
min_distance = 8
```

如果图像大小为 512×512：

```text
peak_threshold = 0.35 ~ 0.45
min_distance = 12 ~ 20
```

得到 marker 点后，给每个 marker 一个独立 ID。

------

## 6.4 Watershed 分割

在 Interior 区域内做 watershed。

流程：

```text
1. 对 interior_bin 计算 distance transform
2. 使用 Seed Heatmap 的 local maxima 作为 markers
3. 在 -distance_map 上执行 watershed
4. mask 限制为 interior_bin
```

输出：

```text
region_label_map
```

每个区域有一个唯一 ID。

------

## 6.5 Connected Components 修正

对 watershed 结果做连通性检查：

```text
1. 每个 label 内部再做 connected components
2. 如果一个 label 被分裂成多个不连通块，则重新编号
3. 删除孤立小区域
```

------

## 6.6 Small-region Removal

删除面积过小的区域。

推荐阈值：

```text
256×256：min_area = 20 ~ 50 pixels
512×512：min_area = 80 ~ 200 pixels
```

也可以按比例设置：

```text
min_area = image_area × 0.0005
```

------

# 10. 评估指标

## 10.1 Interior IoU

评估 Interior Map 是否预测准确：

[
IoU=\frac{P\cap G}{P\cup G}
]

------

## 10.2 Interior Dice

[
Dice=\frac{2|P\cap G|}{|P|+|G|}
]

------

## 10.3 Seed Heatmap MSE

评估热力图回归质量：

[
MSE(\hat{H},H)
]

------

## 10.4 Region-level mIoU

对预测区域和真值区域做匹配，计算平均 IoU。

------

## 10.5 Under-segmentation Rate

衡量多个真实区域是否被预测成一个区域。

例如：

```text
两个真实区域粘成一个预测区域 → 欠分割
```

------

## 10.6 Over-segmentation Rate

衡量一个真实区域是否被切成多个预测区域。

------

## 10.7 Leakage Rate

强烈建议保留这个指标。

定义：

> 模拟 bucket fill 后，错误溢出到目标区域外的像素比例。

可以写成：

[
Leakage=\frac{|\text{Filled Region} - \text{GT Region}|}{|\text{Filled Region}|}
]

这个指标最贴近你的应用场景。

## 最终优点

相比原始方案，它有三个明显优势：

1. **Seed 更容易学习**
   热力图比单像素点监督更适合 U-Net。
2. **Marker 更稳定**
   local maxima 比二值 seed 连通域更适合 watershed。
3. **区域分割更细致**
   可以减少相邻区域粘连，提高快速填色可用性。
