# 数据预处理使用说明

## 1. 当前处理范围

按最新要求，当前脚本仅做一件事（且尽量采用裁剪而不是拉伸）：

- **对原始彩色图像进行裁剪与缩放**，生成统一尺寸的训练输入图。

线稿提取已拆分到独立脚本 `scripts/generate_lineart_anime.py`。

---

## 2. 推荐目录结构

```text
data/
├── raw/
│   └── color/                 # 最原始彩色图像
└── processed/
    └── color_256/             # 裁剪+缩放后的训练输入图
```

---

## 3. 使用命令

```bash
python scripts/prepare_image_inputs.py \
  --input-dir data/raw/color \
  --output-dir data/processed/color_256 \
  --target-size 256
```

---

## 4. 参数说明

- `--target-size`：输出尺寸，最终得到 `target-size x target-size` 的正方形图。
- `--square-tolerance`：判定“接近正方形”的阈值。
  - 满足 `abs(w/h - 1) <= square_tolerance` 时，仍优先先裁成正方形再缩放；
  - 否则执行主体检测裁剪后再缩放。
- `--foreground-threshold`：主体检测灰度阈值（用于框选主体区域）。
- `--bbox-margin-ratio`：主体框外扩比例，避免裁剪过紧。
- `--suffixes`：允许处理的图片后缀。

---

## 5. 处理逻辑

### 5.1 接近正方形图片

- 先裁成正方形再缩放到目标尺寸，避免拉伸变形。

### 5.2 长方形图片

- 灰度阈值检测主体区域；
- 主体框按比例外扩；
- 拟合正方形裁剪框后再缩放。

### 5.3 未检测到主体

- 自动退化为中心裁剪后缩放。

---

## 6. 动漫线稿生成（`netG.pth`）

当你已经得到 `processed/color_512` 后，可用下述脚本批量生成线稿到 `train/images`：

```bash
python scripts/generate_lineart_anime.py \
  --input-dir processed/color_512 \
  --output-dir train/images \
  --model-path "D:/Software/Project/ComfyUI/custom_nodes/comfyui_controlnet_aux/ckpts/lllyasviel/Annotators/netG.pth" \
  --num-workers 4
```

说明：
- `--num-workers` 支持多线程并行加速（CPU 推理建议 2~8 按机器资源调节）。
- 当 `--device cuda` 且 `--num-workers > 1` 时，脚本会自动回落为单线程，避免 CUDA 多线程竞争。
