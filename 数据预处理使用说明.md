# 数据预处理使用说明

## 1. 处理顺序（已按要求调整）

当前预处理流程为：

1. **先处理最原始彩色图像**（裁剪 + 缩放到训练输入尺寸）；
2. 基于预处理后的彩色图像提取**黑白线稿**；
3. 基于线稿提取**区域实例图**（连通域实例 ID）。

对应脚本：`scripts/prepare_image_inputs.py`。

---

## 2. 输出目录建议

```text
data/
├── raw/
│   └── color/                 # 最原始彩色图像
└── processed/
    ├── color_256/             # 预处理后的彩色图像
    ├── lineart_256/           # 从 color_256 提取出的黑白线稿
    └── instances_256/         # 从 lineart_256 提取出的区域实例图（16-bit PNG）
```

---

## 3. 使用命令

```bash
python scripts/prepare_image_inputs.py \
  --input-dir data/raw/color \
  --color-output-dir data/processed/color_256 \
  --lineart-output-dir data/processed/lineart_256 \
  --instance-output-dir data/processed/instances_256 \
  --target-size 256
```

---

## 4. 主要参数说明

- `--target-size`：输出尺寸，最终会得到 `target-size x target-size` 的正方形图。
- `--square-tolerance`：判定“接近正方形”的阈值。
  - 满足 `abs(w/h - 1) <= square_tolerance` 时，直接缩放；
  - 否则执行主体检测裁剪后再缩放。
- `--foreground-threshold`：主体检测灰度阈值（用于框选主体区域）。
- `--bbox-margin-ratio`：主体框外扩比例，避免裁剪过紧。
- `--line-threshold`：从彩色图像提取线稿时的边缘二值化阈值。
- `--min-region-area`：实例提取时保留区域的最小像素面积。

---

## 5. 处理逻辑说明

### 5.1 彩色图像预处理

- 接近正方形：直接缩放到目标尺寸；
- 长方形：
  - 使用灰度阈值检测前景主体框；
  - 对主体框按比例外扩；
  - 拟合正方形裁剪框后再缩放；
- 若未检测到主体：回退为中心裁剪再缩放。

### 5.2 黑白线稿提取

- 对预处理后的彩色图像转灰度；
- 使用边缘响应提取线条；
- 阈值化后输出黑线白底线稿图。

### 5.3 区域实例提取

- 以线稿中的白色可通行区域做 4 邻域连通域搜索；
- 去除触边区域（通常为外背景）；
- 去除面积过小区域；
- 为每个保留区域分配唯一实例 ID（16-bit PNG 保存）。

---

## 6. 注意事项

1. 当前实例图是**自动提取的伪实例标签**，适合做预标注/辅助检查；
2. 若线稿风格差异较大，建议优先调 `--line-threshold` 与 `--min-region-area`；
3. 若主体裁剪不理想，可调 `--foreground-threshold` 与 `--bbox-margin-ratio`。
